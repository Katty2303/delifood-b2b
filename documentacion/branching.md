# Estrategia de Ramificación - GitHub Flow

Para este proyecto utilicé una estrategia simplificada basada en GitHub Flow:
* **main:** Rama principal protegida que contiene el código funcional y estable.
* **Ramas de características (feat-):** Cada integrante trabaja en una rama propia (ej: `feat-backend` o `feat-frontend`) para desarrollar su parte sin romper el código de los demás.
* **Pull Requests (PR):** Al terminar, los cambios se integran a la rama principal mediante una revisión en equipo en GitHub.
